# Work Trees

This directory contains work tree documents that break down daily work into parallelizable tasks for multiple agents.

## Structure

```
docs/work-trees/
├── README.md (this file)
└── YYYY-MM-DD-work-tree.md (daily work trees)
```

## Purpose

Work trees are planning documents that:
- Break down daily objectives into specific tasks
- Assign tasks to appropriate agents
- Identify dependencies and parallelization opportunities
- Track task status and deliverables

## Relationship to Other Documentation

- **Work Trees** (`docs/work-trees/`): Planning and task breakdown
- **Agent Handoffs** (`scripts/YYYY-MM-DD/agent-handoffs/`): Communication between agents
- **Daily Logs** (`docs/logs/YYYY-MM-DD.md`): Summary of what happened

## Usage

1. Create a work tree at the start of each day
2. Break down objectives into parallelizable tasks
3. Assign tasks to agents based on their specialties
4. Update status as work progresses
5. Reference work tree in daily log
