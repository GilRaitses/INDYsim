## Critical Fixes Applied Today

### 1. Track-Level ETI Fix
**Status:** COMPLETE

**Problem:** ETI overflow warnings (4-39 frames per track)
- Global ETI: 23997 frames (camera timestamps, non-uniform)
- Track ETI: 24001 frames (interpolated to uniform 0.05s grid)

**Solution:** Use track's own ETI from `derived_quantities/eti` instead of global ETI
- Perfect match with track data length
- No overflow warnings

### 2. Track-Level LED Extraction
**Status:** COMPLETE

**Problem:** LED NaN values (86 per file) due to camera gaps
- 250ms gap at frame 12286 caused `merge_asof` tolerance failures

**Solution:** Extract LED directly from track's `derived_quantities/led1Val`
- No merge needed (same time grid as track data)
- Zero NaN values

### 3. Onset Detection Threshold Fix
**Status:** COMPLETE

**Problem:** 1289 false onset detections (should be ~40)
- Threshold `led1_max * 0.1 = 25` too sensitive

**Solution:** Use 30% of max diff value as threshold
- `diff_threshold = diff_max * 0.3` (typically ~1500)
- Now detects exactly 40-41 onsets per experiment

### 4. Global time_since_stimulus Recalculation
**Status:** COMPLETE

**Problem:** `time_since_stimulus` max was 1148s (should be ~30s)
- Per-track calculation incorrect for tracks starting mid-experiment

**Solution:** Vectorized global recalculation after track concatenation
- Use `np.searchsorted` for O(n log n) performance
- Now correctly bounded to 0-30s range

### 5. Recursive H5 File Discovery
**Status:** COMPLETE

**Problem:** `glob("*.h5")` only searched top-level directory

**Solution:** Changed to `rglob("*.h5")` for recursive search
