## Critical Issue: Turn Definition Mismatch

**Status:** 🔴 CRITICAL - Turn counts too strict

**Problem:** Current implementation defines turns incorrectly:
- **Current (WRONG):** Turns = reorientations with pauses
- **MAGAT (CORRECT):** Turns = reorientations with numHS >= 1 (at least 1 head swing)

**Evidence from MAGAT Source:**
- `@MaggotReorientation/MaggotReorientation.m`: "reorientations have zero or more headSwings"
- `load_multi_data_gr21a.m` line 169: `turnStartTime = reorientation with numHS >= 1`
- `playMovie.m` lines 324-327: If numHS == 0 → "pause", if numHS > 0 → "turn"

**Impact:**
- Turn counts are severely underestimated
- Many valid turns (with head swings but no pauses) are missed
- Turn rates don't match MATLAB reference

**Fix Applied:**
- Changed turn definition to match MAGAT: turns = reorientations with numHS >= 1
- Updated turn extraction code to count head swings within reorientations
- See `docs/logs/2025-11-14/MAGAT_TURN_DEFINITION_ANALYSIS.md` for full analysis
