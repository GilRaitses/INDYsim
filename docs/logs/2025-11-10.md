## Technical Details

### MATLAB Reference Implementation
- **File:** `ExperimentAggregator.m` (lines 338, 352)
- **Method:** Count reorientation times that fall within bin boundaries
- **Formula:** `turn_rate = (reorientations / duration_seconds) * 60`

### Python Fix
- **Before:** `track_reorientations = track_bin_events['is_reorientation'].sum()`
- **After:** `bin_reorientations = np.sum((track_reo_times >= bin_start) & (track_reo_times < bin_end))`

### Key Difference
- MATLAB: Counts actual reorientation events
- Python (old): Summed boolean flags from aggregated bins → miscounts
- Python (new): Counts reorientation times directly → matches MATLAB

### Temporal Alignment Fixes (Final Session)
- **Problem:** Temporal drift causing spikes to appear BEFORE t=0 (around t=-3 to -4 seconds)
- **Root Cause:** Time binning used 'mean' aggregation, causing temporal drift
- **Solution:**
  - Changed time aggregation from 'mean' to 'first' in `create_event_records()` (engineer_dataset_from_h5.py)
  - Added support for trajectories CSV (frame-level accuracy) in turn rate calculation
  - Uses first frame time in each bin instead of averaging to avoid drift
- **Files Modified:**
  - `scripts/engineer_dataset_from_h5.py` (lines 1000, 1006): Changed 'time': 'mean' → 'time': 'first'
  - `scripts/create_eda_figures.py` (lines 181-273): Added trajectories_df parameter for frame-level accuracy
